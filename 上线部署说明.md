# 剧本资产分析网站上线部署说明

## 推荐方案

推荐使用一台云服务器部署，例如阿里云 ECS、腾讯云 CVM、华为云服务器等。

这个项目不建议直接部署到纯前端平台，因为它需要：

- Node 后端接口
- Word/PDF 文件上传
- 较长时间的 AI 分析任务
- 本地历史记录数据
- 私密 API Key

## 上线前准备

服务器建议配置：

- 2 核 CPU / 4GB 内存起步
- 系统：Ubuntu 22.04 或更新版本
- Node.js：20 LTS 或更新版本
- 开放端口：80、443

正式上线前请修改 `.env`：

```ini
QWEN_API_KEY=你的模型平台API密钥
QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
QWEN_MODEL=qwen-plus
QWEN_MODEL_OPTIONS=qwen-plus,qwen-max,qwen-turbo,qwen-long
ANALYSIS_CHUNK_CHARS=40000
QWEN_TIMEOUT_MS=180000
QWEN_CONCURRENCY=2
PORT=3003
APP_ENV=production
TRUST_PROXY=true
APP_USERNAME=你的登录账号
APP_PASSWORD=至少12位强密码
```

不要把 `.env` 发给别人，也不要提交到公开仓库。

生产环境会拒绝使用默认账号密码；如果 `APP_USERNAME=admin` 或 `APP_PASSWORD=123456`，服务会直接启动失败。

## 部署步骤

1. 把项目上传到服务器，例如 `/opt/script-asset-analyzer`。

2. 进入项目目录：

```bash
cd /opt/script-asset-analyzer
```

3. 安装依赖：

```bash
npm install
```

4. 构建前端：

```bash
npm run build
```

5. 测试启动：

```bash
npm start
```

打开：

```text
http://服务器IP:3003
```

如果能看到登录页，说明项目本身已经跑起来。

## 常驻运行

建议用 PM2 让网站常驻运行：

```bash
npm install -g pm2
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup
```

以后常用命令：

```bash
pm2 status
pm2 logs script-asset-analyzer
pm2 restart script-asset-analyzer
pm2 stop script-asset-analyzer
```

后续更新可以直接执行：

```bash
bash scripts/deploy.sh
```

更多维护细节见 `维护与更新流程.md`。

## 绑定域名

安装 Nginx：

```bash
sudo apt update
sudo apt install nginx
```

新建配置：

```bash
sudo nano /etc/nginx/sites-available/script-asset-analyzer
```

写入：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    client_max_body_size 50m;

    location / {
        proxy_pass http://127.0.0.1:3003;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/script-asset-analyzer /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## HTTPS

域名解析到服务器后，建议用 Certbot 配 HTTPS：

```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

生产环境登录 Cookie 会启用 `Secure`，所以线上必须通过 HTTPS 访问。

## 数据和备份

历史记录默认保存在：

```text
data/app-data.json
```

上线后要定期备份 `data/` 目录。  
如果后续多人使用，建议改成数据库，例如 SQLite、PostgreSQL 或 MySQL。

## 上线后的注意事项

- 一定要修改默认账号和密码。
- 不要公开 API Key。
- 如果剧本内容敏感，必须使用 HTTPS。
- 大剧本分析时间较长，Nginx 需要保留较长超时时间。
- 免费平台或 Serverless 平台可能会因为超时、无持久磁盘、文件上传限制而不稳定。
