# 维护与更新流程

## Git 仓库

本项目应该提交到 Git 仓库，但不要提交这些内容：

- `.env`
- `data/`
- `backups/`
- `logs/`
- `node_modules/`
- `dist/`

这些已经写入 `.gitignore`。

当前电脑命令行里没有检测到 Git。如果还没安装 Git，请先安装：

```text
https://git-scm.com/download/win
```

安装后在项目目录执行：

```bash
git init
git add .
git commit -m "Initial deployable version"
```

如果你有 GitHub / Gitee / GitLab 仓库：

```bash
git remote add origin 仓库地址
git branch -M main
git push -u origin main
```

## 推荐部署方式：PM2

服务器首次部署：

```bash
cd /opt
git clone 仓库地址 script-asset-analyzer
cd script-asset-analyzer
cp .env.example .env
nano .env
npm install
npm run build
npm install -g pm2
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup
```

之后每次更新：

```bash
cd /opt/script-asset-analyzer
bash scripts/deploy.sh
```

`scripts/deploy.sh` 会自动：

- 拉取最新代码
- 备份 `data/`
- 安装依赖
- 跑测试
- 构建前端
- 重启 PM2

## 备份历史数据

手动备份：

```bash
bash scripts/backup-data.sh
```

默认会备份到：

```text
backups/data-年月日-时分秒.tar.gz
```

默认保留 14 天备份。

服务器定时备份可以执行：

```bash
crontab -e
```

加入：

```cron
0 3 * * * cd /opt/script-asset-analyzer && bash scripts/backup-data.sh >> logs/backup.log 2>&1
```

## Docker 部署方式

如果你使用 Docker 或 1Panel，可以使用：

```bash
docker compose up -d --build
```

更新：

```bash
git pull
docker compose up -d --build
```

Docker 会把本地 `data/` 挂载到容器里的 `/app/data`，历史记录不会因为容器重建而丢失。

## 线上修 Bug 的固定流程

建议之后统一按这个流程：

1. 本地修复代码
2. 本地运行测试：

```bash
npm test
npm run build
```

3. 提交 Git：

```bash
git add .
git commit -m "Fix xxx"
git push
```

4. 服务器更新：

```bash
cd /opt/script-asset-analyzer
bash scripts/deploy.sh
```

## 重要提醒

- `.env` 只放在服务器，不进 Git。
- `data/` 是用户历史记录，一定要备份。
- 当前 JSON 存储适合个人或小团队使用；如果多人同时大量使用，建议后续升级数据库。
- PM2 当前固定单进程运行，避免多个进程同时写 JSON 文件。
